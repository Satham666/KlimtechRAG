## [HIGH #1] SSRF w `/web/fetch` bez blokady adresów prywatnych `[NOWE]`

```
[PLIK]    backend_app/routes/web_search.py:158
[PROBLEM] Weryfikacja URL sprawdza tylko prefix http/https.
          Nie blokuje adresów: 192.168.x.x, 10.x.x.x, 127.0.0.1,
          169.254.x.x (link-local), ::1 (IPv6 loopback).
          Endpoint nie wymaga auth.
[RYZYKO]  Atakujący może używać backendu jako proxy do skanowania
          sieci wewnętrznej 192.168.31.0/24:

          POST /web/fetch {"url": "http://localhost:5678"}  — n8n
          POST /web/fetch {"url": "http://192.168.31.1:22"} — router
[FIX]     Dodać walidację przed requestem:
          import ipaddress, socket
          from urllib.parse import urlparse

          host = urlparse(request.url).hostname
          try:
              ip = ipaddress.ip_address(socket.gethostbyname(host))
              if ip.is_private or ip.is_loopback or ip.is_link_local:
                  raise HTTPException(400, "Private/local addresses not allowed")
          except socket.gaierror:
              raise HTTPException(400, "Cannot resolve host")
```
